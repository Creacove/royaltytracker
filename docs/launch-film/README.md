# RoyaltyTracker Launch Film Package

This folder contains the working package for the first `30-45s` product launch film.

The package is built around the current app surface:

- `Reports & Statements`
- `Dashboard`
- `AI Insights`
- `Artist Snapshot`
- `Track Snapshot`
- `Data Quality Queue`
- `Transactions`

## Files

- `storyboard.md`: timed shot list, edit rhythm, and route-by-route capture plan
- `capture-runbook.md`: exact capture setup, browser framing, and shot execution notes
- `demo-content.json`: staged demo data, typed prompts, answer copy, and UI content targets
- `overlay-copy.srt`: subtitle-style overlay timing for the motion edit

## Production Rules

- Format: `16:9`, master at `3840x2160`, deliver downscaled social export as needed
- Runtime target: `42s`
- Tone: `editorial luxury`
- Meaning carrier: `on-screen text only`
- Footage source: `real app UI with staged demo content`
- Music: `one premium editorial bed`, no voiceover

## Film Promise

The piece should make one idea obvious within the first `10s`:

`Royalty data becomes instant decisions.`

## Demo Scope

The staged workspace should look like a serious publisher account, not a sandbox. Use the exact prompts, names, and figures in `demo-content.json` unless a stronger on-screen composition requires a small visual adjustment.
